Presentation des technologies utilisées

Pour les differents projets informatiques de diagamter et aleacontroles, plusieurs technologies sont utilisées, pour le coté front-end , donc ce qui s'affiche pour l'utilisateur,le coté backend, ce qui se passe coté serveur, et pour les logiciels.

Pour les sites web, le developpement est fait en PHP, un langage de programmation tres utilisé pour les sites web. Cela a evolué avec le temps, notamment par l'utilisation d'un framework : Laravel. laravel est un framework basé sur le modele MVC, (modèle-vue-controller) qui permet de structurer le code de facon plus claire et plus lisible. Le paterne MVC est un paterne qui utilise la programmation orientée objet (POO), pour séparer le code du site en trois parties.

La premiere est le modèle, qui permet de definir pour chaque donnée differente, ce qu'il est possible de faire avec et de modifier. Ces données correspondent dans notre cas à nos differentes tables dans la base de données, et les modèles rassemblent donc diffrentes methodes pour récuperer, modifier ou supprimer des données.

La deuxieme partie est la vue, qui correspond à ce que l'utilisateur voit sur le site. Ici on utilise un systeme de templates blade, qui permet de creer des bouts de codes qui vont s'inserer dans plusieurs autres template, comme un header ou un footer, mais aussi de gerer la creation de page avec des données variables, des conditions ou des boucles, qui sont envoyées par le controlleur.

La derniere partie est donc le controlleur, qui va gerer les requetes de l'utilisateur, et renvoyer les bonnes vues avec les bonnes données. Chaque fonction de nos controlleurs correspond à une route, qui est une url rentrée par l'utilisateur, et va faire le lien avec les modèles et renvoyer une vue avec des variables qui vont venir afficher le bon contenu à l'utlisateur.

Laravel prend en charge beaucoup d'autres aspects d'un service web. Il existe un principe de middleware, qui permet de gerer les requetes avant qu'elles n'arrivent au controlleur, pour verifier par exemple si l'utilisateur est connecté, ou si il a les droits pour acceder à une page. Il existe un ORM, qui permet de gerer les relations entre les tables de la base de donnée, et de creer des requetes plus facilement, mais qui gere aussi un systeme de migrations, pour venir modifier les structures de notre BDD, de maniere propre et simple. Il existe également des commandes, un systeme de cron, ...

Pour la partie front-end, on utilise avec les blade php du scss, un preprocesseur css, qui permet de créer des variables, des fonctions,... et de les compiler en css, mais également boostrap, modifié pour correspondre à la DA diagamter. On utilise également du javascript, principalement avec jquery, mais également plusieurs plugins, comme datatables ou fullcalendar, pour ameliorer l'experience utilisateur.

On utilise également differentes API, comme smsMode et mailjet, respectivement pour de l'envoie de SMS et de l'envoie de mail, mais aussi google maps et openstreetmap pour les cartes.

On utilise aussi elasticsearch, un moteur de recherche, pour permettre à l'utilisateur de trouver plus facilement des informations sur le site.

Pour les applications, on utilise du VBA pour la plus ancienne, et WPF pour la plus recente, qui est un framework pour creer des applications windows, a laquel on rajoute telerik, un framework pour creer des interfaces graphiques plus facilement.

Par Nathan Bertaud
